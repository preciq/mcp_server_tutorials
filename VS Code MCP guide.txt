This guide only covers **connecting MCP servers to VS Code** (Copilot Chat / Agent mode).

Important:
* This repo does **not** include MCP servers.
* If you don’t have a server yet, VS Code will show errors like `Cannot find module '/absolute/path/to/your-mcp-server.js'`. That is expected until you point `mcp.json` at a real server script.

---

## 0) Prereqs (VS Code side)

* **VS Code 1.99+** (that’s when MCP-in-Copilot support is documented). ([GitHub Docs][1])
* You’ll use a workspace config file: **`<repo-root>/.vscode/mcp.json`**. ([GitHub Docs][1])
* MCP servers should use **stdio transport**, which VS Code supports. ([Visual Studio Code][2])

---

## 1) Configure your servers inside VS Code (workspace MCP config)

Create this file:

**`<repo-root>/.vscode/mcp.json`** ([GitHub Docs][1])

Example config (Linux/macOS paths; replace the placeholders with real server scripts):

```json
{
  "servers": {
    "my-server-js": {
      "type": "stdio",
      "command": "node",
      "cwd": "${workspaceFolder}",
      "args": ["/absolute/path/to/your-mcp-server.js"]
    },
    "my-server-py": {
      "type": "stdio",
      "command": "/absolute/path/to/venv/bin/python",
      "cwd": "${workspaceFolder}",
      "args": ["/absolute/path/to/your-mcp-server.py"]
    }
  }
}
```

Key points:

* VS Code MCP config is primarily `"servers": { ... }` (and optionally `"inputs": [...]` for secrets). ([Visual Studio Code][2])
* Don’t start stdio servers via wrappers that print to **stdout** (e.g., `npm run ...`). For MCP stdio, stdout is the protocol channel—logs should go to stderr. (These quickstart servers already log to stderr, but package managers can add stdout noise.)

**Windows tweak (Python):**

```json
"command": "${workspaceFolder}\\mcp-client-python\\.venv\\Scripts\\python.exe"
```

---

## 2) Start + verify in VS Code

1. Open **Command Palette** → run **`MCP: List Servers`** to confirm they appear. ([GitHub Docs][1])
2. Open **Copilot Chat**, switch to **Agent** mode (tools-enabled). ([GitHub Docs][1])
3. Use the tools picker to confirm you see tools from your configured servers.

If you don’t have a server yet:
* Leave `.vscode/mcp.json` empty or remove the placeholder servers.
* Expect connection errors until you add a real server script path.

For how to build a server: https://modelcontextprotocol.io/docs/develop/build-server
